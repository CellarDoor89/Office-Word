# DocCheck — проверка и автоисправление документа по «Памятке»

Один инструмент в нескольких обличиях: **GUI-версия с перетаскиванием файла** (для обычных пользователей) и три варианта-«движка» под капотом (VBA / PowerShell / Python).

Все варианты делают одно и то же: сначала проверяют документ, затем (по согласию пользователя) исправляют то, что можно исправить автоматически. Что нельзя — выводят списком с пометкой «ручная правка».

---

## 🚀 Самый простой способ — GUI (DocCheck.exe)

**Для конечного пользователя.** Программа собрана в **один .exe-файл**, в котором уже всё, что нужно — Python ставить НЕ требуется. Просто кладёте `DocCheck.exe` рядом с `blank_template.txt` куда угодно (рабочий стол, сетевая папка, флешка) и запускаете.

### Как пользоваться

1. Запустить `DocCheck.exe` — откроется окно.
2. **Перетащить** файл `.docx` в большое окно посередине (или нажать «Выбрать файл…»).
3. Появится список замечаний с пометками:
   - `[можно исправить]` — оранжевые, чинятся автоматически;
   - `[ручная правка]` — красные, надо править руками в Word.
4. Нажать **«Исправить автоисправимые»** — рядом с исходным файлом появится копия `имя_fixed.docx`. Перепроверка покажет, что осталось.
5. Кнопка **«Открыть исправленный файл»** откроет результат в Word.

Чтобы поменять реквизиты бланка под свою организацию — нажмите **«Открыть шаблон бланка…»**, файл откроется в Блокноте.

### Где взять .exe

* **Готовый .exe** автоматически собирается на каждый коммит и доступен на странице репозитория: **Actions → Build DocCheck.exe → последний зелёный прогон → Artifacts → `DocCheck-windows`**. Внутри ZIP — `DocCheck.exe` + `blank_template.txt`.
* **Собрать самому** (если есть Python 3.10+ на любой Windows-машине): `cd doc_validator\gui && build.bat`. На выходе `dist\DocCheck.exe` (~20 МБ).

### Особенности и ограничения

* **Антивирус / SmartScreen.** PyInstaller-овские .exe иногда триггерят «неизвестный издатель». Если SmartScreen ругается — «Подробнее → Выполнить в любом случае». Если корпоративный антивирус блокирует — нужно либо подписать .exe вашим сертификатом, либо попросить ИТ добавить в whitelist.
* **Первый запуск медленнее** (5–10 сек) — .exe распаковывает себя во временную папку. Дальше быстрее.
* **Только .docx.** Старые `.doc` не читаются: откройте в Word и сохраните как `.docx`.
* **Drag-and-drop** работает из Проводника. Если запустить .exe от имени администратора, перетаскивание из обычного окна Проводника может не работать (разные уровни прав) — лучше запускать без админа.

## Что проверяется и что чинится

| Пункт Памятки | Проверка | Автоисправление |
|---|---|---|
| 1. Поля 30/20/20/10 мм | да (с учётом бланка) | **да** — задаёт точные значения |
| 2. Шрифт Times New Roman | да | **да** — переключает шрифт во всех runs |
| 2. Кегль 13–14 пт | да | **да** — выставляет 14, если вне диапазона |
| 2. Межстрочный 1.0–1.5 | да | **да** — выставляет 1.5 |
| 2. Абзацный отступ 1.25 см | да | **да** — для основного текста (не заголовков) |
| 3. Нумерация страниц | да | **да** — поле PAGE сверху по центру |
| 4. Заголовок «О …»/«Об …» под адресатом | да | нет — ручная правка |
| 6. Отметка об исполнителе | да | нет (это контент) — ручная правка |
| 8. Гриф ДСП + № экземпляра (правый верхний угол) | да | нет — ручная правка |
| 9. Блок «Адресат» (справа, ФИО или организация) | да | нет — ручная правка |
| 9. Адресат — выравнивание строк по левому краю блока | да | нет — ручная правка |
| 9. Адресат — межстрочный 1.0 | да | нет — ручная правка |
| 9. Формат инициалов «Фамилия И.О.» | да | нет — ручная правка |
| Бланк организации (левая часть шапки) | да, по `blank_template.txt` | нет — ручная правка |

Целевые значения при исправлении вынесены в константы (`FIX_FONT_SIZE`, `FIX_LINE_SPACING`) — поменяйте, если в организации принят другой стандарт.

## Безопасность исходного файла

* **VBA:** правит активный документ, но **не сохраняет**. Если результат не нравится — Ctrl+Z откатит изменения, или просто закрыть без сохранения. Если устраивает — Ctrl+S.
* **PowerShell:** по умолчанию сохраняет исправленный документ в **копию `имя_fixed.docx`**. С флагом `-InPlace` пишет в исходник, предварительно создав `.bak`.
* **Python / GUI:** так же — копия `имя_fixed.docx` по умолчанию; с `--in-place` (только в CLI-режиме) пишет в исходник и делает `.bak`.

## Под капотом — три варианта движка (для администратора / разработчика)

Это альтернативы GUI-обёртке: тот же набор проверок, но запускаются вручную. Подходит, если по корпоративной политике .exe запретили, а Word или PowerShell — нет.

## Вариант 1 — VBA-макрос

### Установка (один раз)

1. Word → **Alt + F11** → редактор VBA.
2. Слева выбрать `Normal` (это `Normal.dotm`).
3. **File → Import File…** → выбрать `vba/DocCheck.bas`.
4. Закрыть редактор VBA.

### Использование

1. Открыть документ в Word.
2. **Alt + F8 → DocCheck.CheckDocument → Run.**
3. В окне появится список замечаний с пометками `[можно исправить]` / `[ручная правка]`.
4. На вопрос «Исправить?» нажать **Да**.
5. Просмотреть результат, **Ctrl + S** для сохранения (или **Ctrl + Z** для отката).

Для запуска в один клик: повесить макрос на кнопку панели быстрого доступа — **Параметры Word → Настроить ленту / Панель быстрого доступа → Макросы → DocCheck.CheckDocument**.

## Вариант 2 — PowerShell + COM Word

```powershell
# только проверить
powershell -ExecutionPolicy Bypass -File powershell\Check-WordDoc.ps1 -Path "C:\письмо.docx"

# проверить и исправить (сохранится письмо_fixed.docx рядом)
powershell -ExecutionPolicy Bypass -File powershell\Check-WordDoc.ps1 -Path "C:\письмо.docx" -Fix

# переписать исходный файл (с .bak)
powershell -ExecutionPolicy Bypass -File powershell\Check-WordDoc.ps1 -Path "C:\письмо.docx" -Fix -InPlace
```

Без `-Path` — появится диалог выбора файла. Удобно сделать ярлык на рабочем столе или `.cmd`-обёртку.

## Вариант 3 — Python + python-docx

```bash
pip install python-docx
```

```bash
python python/check_word_doc.py "путь/к/документу.docx"             # только проверка
python python/check_word_doc.py "путь/к/документу.docx" --fix       # проверка + спросит и исправит
python python/check_word_doc.py "путь/к/документу.docx" --fix --yes # без вопросов
python python/check_word_doc.py "путь/к/документу.docx" --fix --in-place  # переписать исходник (.bak)
```

Коды возврата: `0` — всё ОК, `1` — есть замечания (или остались после исправления), `2` — файл не найден.

## Сравнение

| | VBA | PowerShell | Python |
|---|---|---|---|
| Нужен Word на машине | да | да | **нет** |
| Что-то ставить | нет | нет | да (python + python-docx) |
| Запуск | кнопка в Word | ярлык/cmd | командная строка |
| Откат правок | **Ctrl+Z** в Word | из `.bak` или `_fixed` копия | из `.bak` или `_fixed` копия |
| Корректный подсчёт страниц | **да** | **да** | эвристика (≥40 абзацев) |
| Шанс пройти политики | средний (макросы) | средний (PS-политика) | низкий (нужны права на pip) |

**В вашем сценарии (нельзя ставить ПО, есть ЭЦП):** оптимально — **VBA**. Word уже стоит, макрос можно подписать корпоративным сертификатом, изменения применяются в открытом документе и откатываются стандартным Ctrl+Z.

## Настройка бланка под свою организацию

Левая часть шапки проверяется по файлу **`doc_validator/blank_template.txt`** — это просто текстовый файл со списком ожидаемых строк бланка. Сейчас он предзаполнен реквизитами **УФНС России по Ямало-Ненецкому автономному округу** (из вашего файла «общий бланк.doc»). Если что-то поменяется (новый адрес, телефон, переподчинение) — откройте файл в Блокноте и поправьте строки.

Синтаксис:

```
# строки с # — комментарии, пустые игнорируются

Название организации              # ищется как подстрока (с учётом регистра)
?ЗАМЕСТИТЕЛЬ РУКОВОДИТЕЛЯ         # ? — необязательно (только предупреждение)
re:Телефон:\s*\(\d{3,5}\)\s*\d    # re: — регулярное выражение
?re:Факс:\s*.+                     # можно сочетать
```

Где скрипты ищут файл:

* **VBA** — путь задан константой `BLANK_TEMPLATE_PATH` в начале `DocCheck.bas` (по умолчанию `C:\Users\Public\Documents\blank_template.txt`). Поменяйте константу или положите файл по этому пути.
* **PowerShell** — рядом со скриптом. Можно явно: `-BlankTemplate "C:\путь\blank_template.txt"`.
* **Python** — рядом со скриптом или в `doc_validator/`. Можно явно: `--blank-template путь`.

Если файла нет — проверка бланка тихо пропускается.

## Прочие пороги

Все три скрипта читают пороги из констант в начале файла. Если в организации, например, допустим кегль 12–14, поменяйте:

* VBA: блок `' ----- параметры из Памятки -----` в `DocCheck.bas`
* PowerShell: блок `# ---------- параметры из Памятки ----------` в `Check-WordDoc.ps1`
* Python: блок `# ---------- параметры из Памятки ----------` в `check_word_doc.py`

## Проверено

* На «Памятке» Python-вариант ловит 4 отклонения по форматированию, после `--fix` перепроверка чистая.
* На «общий бланк.doc» (после конвертации) бланк УФНС-ЯНАО распознаётся полностью — все 9 строк найдены.
* На «Письмо по работе с новыми ЭС_2.doc» (бланк центральной ФНС, не наш) проверка корректно отмечает все 9 ожидаемых строк УФНС-ЯНАО как отсутствующие — без ложных срабатываний.
* На синтетическом письме с другим бланком (ООО «Ромашка») все 9 ожидаемых строк отмечаются как отсутствующие.
* Edge-cases по инициалам: «ЕгоровуД.В.» (слитно) и «Егорову Д.» (один инициал) ловятся корректно.
* GUI: импорт модулей и привязка к ядру валидатора прогнаны на Linux со mock-tkinter — на тестовом документе возвращает идентичный CLI-варианту результат.

## Особый случай: бланк с гербом на первой странице

В шаблонах с гербом (например, бланк УФНС) верхнее поле специально уменьшено — иначе герб не помещается. Проверка это понимает: если документ начинается с таблицы (типичная разметка бланка), **верхнее поле не проверяется**. Левое / нижнее / правое поля проверяются как обычно.

Признак «бланк»: самый первый непустой элемент документа — таблица. Без эвристик на конкретный герб или название организации, чтобы работало в любой организации.

## Ограничения проверок шапки

* Шапка считается как **первые ~60 параграфов документа** (включая ячейки таблиц). Если в организации шапка длиннее — поменяйте константу `limit`/`LIMIT` в скриптах.
* Правый блок определяется эвристически: выравнивание по правому краю, либо левый отступ > 7 см, либо параграф в последней ячейке строки таблицы. Нестандартные схемы (например, шапка через надпись/текстовое поле) могут не распознаться.
* Проверка бланка — это поиск подстрок/регулярок, а не побайтовое сравнение шрифтов/размеров/расположения. Если важно проверять и форматирование бланка тоже — приложите шаблон .dotx, расширим.
* Проверка падежа в адресате (дательный vs именительный) и точной формулировки заголовка («О чём?») не делается — это требует морфологического анализа.
