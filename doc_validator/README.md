# DocCheck — проверка оформления документа по «Памятке»

Три варианта одного и того же набора проверок. Выбирайте по тому, что разрешено в организации.

## Что проверяется

| Пункт Памятки | Проверка |
|---|---|
| 1. Поля | левое 30, верхнее 20, нижнее 20, правое 10 мм (допуск ±0.5 мм) |
| 2. Шрифт | Times New Roman |
| 2. Кегль | 13 или 14 пт |
| 2. Межстрочный | 1.0 – 1.5 |
| 2. Абзацный отступ | 1.25 см (для основного текста, не заголовков) |
| 3. Нумерация страниц | если страниц ≥ 2 — поле PAGE сверху по центру, без «-», «стр.», «с.» |
| 6. Отметка об исполнителе | телефон в формате `8(код)номер` |
| 8. Гриф ДСП | если есть «Для служебного пользования», должен быть «Экз. № …» |

Чего проверки **не** делают: семантику заголовка («О чём?»), правильность падежей в адресате, согласование в СЭД. Это всё равно нужно смотреть глазами.

## Вариант 1 — VBA-макрос (рекомендуется)

**Плюсы:** ставить ничего не надо, работает внутри Word, кнопкой за 1 клик.
**Минусы:** в организации может быть отключено выполнение макросов — придётся попросить ИТ включить или подписать макрос корпоративным сертификатом.

### Установка (один раз)

1. Открыть Word, нажать **Alt + F11** — откроется редактор VBA.
2. В дереве слева выбрать `Normal` (это `Normal.dotm`, глобальный шаблон пользователя).
3. Меню **File → Import File…**, выбрать `vba/DocCheck.bas`.
4. Закрыть редактор VBA. Сохранение произойдёт автоматически при закрытии Word.

### Использование

* Открыть проверяемый документ.
* **Alt + F8 → DocCheck.CheckDocument → Run.**
* Появится окно с отчётом «OK» или списком замечаний.

Чтобы было ещё удобнее — в Word: **Параметры → Настроить ленту / Панель быстрого доступа → Макросы → DocCheck.CheckDocument** — повесить на кнопку.

## Вариант 2 — PowerShell + COM Word

**Плюсы:** не трогает Normal.dotm, удобно гонять по нескольким файлам из проводника / по расписанию. Использует тот же Word, что у вас и так стоит.
**Минусы:** часто отключена политика выполнения `.ps1` (нужен флаг `-ExecutionPolicy Bypass` или подпись).

### Использование

Запуск из командной строки:

```powershell
powershell -ExecutionPolicy Bypass -File powershell\Check-WordDoc.ps1 -Path "C:\Документы\письмо.docx"
```

Без `-Path` появится диалог выбора файла.

Можно сделать ярлык на рабочем столе:

* **Объект:** `powershell.exe -ExecutionPolicy Bypass -File "C:\путь\Check-WordDoc.ps1"`
* двойной клик → откроется диалог → выбираете .docx → видите отчёт в консоли.

Чтобы не мелькала консоль, можно завернуть в `.cmd`:

```cmd
@echo off
powershell -ExecutionPolicy Bypass -File "%~dp0Check-WordDoc.ps1" %*
pause
```

## Вариант 3 — Python + python-docx

**Плюсы:** работает без Word вообще (читает `.docx` как ZIP), легко встроить в CI / pre-commit / любую автоматизацию.
**Минусы:** на рабочей машине обычно нет Python — отпадает в полностью закрытом контуре. И часть проверок (например, реальный подсчёт страниц) ограничена, потому что без движка Word странички не пересчитаешь — используется эвристика по количеству абзацев.

### Установка

```bash
pip install python-docx
```

### Использование

```bash
python python/check_word_doc.py "путь/к/документу.docx"
```

Код возврата: `0` — всё ОК, `1` — есть замечания, `2` — файл не найден.

## Сравнение

| | VBA | PowerShell | Python |
|---|---|---|---|
| Нужен Word на машине | да | да | **нет** |
| Что-то ставить | нет | нет | да (python + python-docx) |
| Запуск | кнопка в Word | ярлык / cmd | командная строка |
| Корректный подсчёт страниц | **да** | **да** | эвристика |
| Подходит для массовой проверки | средне | **хорошо** | **хорошо** |
| Шанс пройти политики безопасности | средний (макросы) | средний (PS) | низкий (нужны права на установку) |
| Кому проще объяснить | пользователю Word | админам | разработчику |

**Практический совет:** в закрытом контуре с большей вероятностью пройдёт VBA — Word у всех уже стоит, и макрос можно подписать тем же ЭЦП-сертификатом, что используется для подписи документов. PowerShell тоже без установки, но политика выполнения скриптов в организациях обычно строже политики макросов. Python — для тех, у кого есть рабочая станция разработчика или централизованная сборка.

## Настройка под организацию

Все три скрипта читают пороги из констант в начале файла. Если в вашей организации, например, допустим кегль 12–14, поменяйте:

* VBA: блок `' ----- допуски -----` в `DocCheck.bas`
* PowerShell: блок `# ---------- параметры из Памятки ----------` в `Check-WordDoc.ps1`
* Python: блок `# ---------- параметры из Памятки ----------` в `check_word_doc.py`
